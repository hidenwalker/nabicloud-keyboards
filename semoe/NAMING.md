# 세모이 자판 명칭·표시명 (NabiCloud 세모이 애드온)

이 문서는 세모이(세벌식 모아치기 e) 자판의 **본가 정식명 ↔ 나비구름 표기**의 대응과
충실도, 그리고 설정창에 보이는 **표시명을 사용자가 바꾸는 방법**을 정리한다.

본가 = 신세기(Sinseiki) / ssgi.kr. 데이터 라이선스 CC BY-SA 4.0, 본가 릴리스 **v260615** 기준.
동작은 본가 날개셋 chord 엔진과의 일치를 보증하지 않는 베스트 에포트 유사 동작이다.

---

## 1. 명칭 충실도표 (본가 v260615 대비)

| 본가 정식명 | 나비구름 표기(자판 id) | 본가 일치도 | 해석 |
|---|---|---|---|
| 세벌식 모아치기 2014 | 세벌식 세모이 2014 (`3moa-semoe-2014`) | ≈65% | pre-e 구세대 (그래서 낮음 — 정상) |
| 세벌식 모아치기 2015 | 세벌식 세모이 2015 (`3moa-semoe-2015`) | ≈92% | pre-e, e로 수렴 중 |
| 세벌식 모아치기 e-2016 | 세벌식 세모이 2016 (`3moa-semoe-2016`) | ≈100% | e-시리즈 확장 기호열 |
| 세벌식 모아치기 e-2017 | 세벌식 세모이 2017 (`3moa-semoe-2017`) | ≈100% | e-시리즈 = 현 세모이 |
| 세벌식 모아치기 e-2018 | 세벌식 세모이 2018 모아치기 (`3moa-semoe-2018`) | ≈100% | e-시리즈 현행(v260615) |

- "일치도"는 v260615 의 자모 배열·결합표 대비 본 변환본의 일치 비율(자동 대조 + 규격 PDF 정독)이다.
  2014/2015 가 낮은 것은 본가 자체가 e-시리즈 이전 세대라 배열이 다르기 때문이며 **오류가 아니다**.
- 나비구름은 일관성을 위해 "세벌식 세모이 YYYY" 표기를 쓴다. 본가 정식명(e-표기)과는 위 표로 대응한다.

## 2. 표시명에 버전(v260615) 부기

`v260615` 는 **개별 자판의 버전이 아니라 본가 저장소(Sinseiki/Semo-e_keyboard) 스냅샷 버전**이다
(5종 모두 이 한 스냅샷에서 옮김 — §1·NOTICE-reference.md). 따라서 모든 자판명에 같은 값을 붙이면
중복·혼란이라, 대표 자판인 **2018 모아치기 표시명에만** 참고로 표기한다(예:
`세벌식 세모이 2018 모아치기 (v260615)`). 나머지 4종은 연도명만 둔다. 이는 자판 XML 의 `<name>`
을 바꾸는 것이 아니라 아래 §3 의 표시명 오버라이드 테이블로 표시 단계에서만 덧붙인다(동작·id 불변).
사용자가 원하면 §3 파일에서 다른 자판에도 직접 버전을 붙일 수 있다.

## 3. 표시명 오버라이드 테이블 (사용자 수정 가능)

표시명은 config.ini 와 **별개**로 `<나비구름 설치폴더>\keyboards\keyboard-names.txt` 한 파일이
결정한다. 형식은 한 줄에 `<자판id> = <보이고 싶은 표시명>` 이며 `#`/`;` 주석과 빈 줄은 무시된다.

```
3moa-semoe-2018  = 세벌식 세모이 2018 모아치기 (v260615)
2                = 두벌식(내 표기)
```

- 이 파일은 **본체 인스톨러**가 설치한다(외부 XML 표시명은 세모이 외 자판에도 쓰이므로 본체가 소유).
- **세모이 애드온 인스톨러**는 설치 시 이 파일이 있으면 그 뒤에 세모이 5종 라인(버전명 포함)을
  `nabicloud-semoe-addon` 마커 블록으로 **append** 한다(이미 있으면 중복 추가하지 않음).
- 사용자가 `=` 뒤를 원하는 이름으로 직접 편집하면 그대로 반영된다(같은 id 가 여러 번이면 마지막 줄 우선).
- 변경은 입력기를 한 번 재활성화(또는 호스트 앱 재시작)하면 목록에 반영된다.
- 애드온을 제거해도 append 된 라인은 남지만, 해당 자판 XML 이 사라져 그 id 는 목록에 없으므로
  표시에 영향이 없다(사문 라인). 깔끔히 지우려면 `nabicloud-semoe-addon` 블록을 직접 삭제하면 된다.

> 동작 원리: 셸(NabiCloud.cpp `LoadNameOverridesOnce`/`BuildKeyboardListOnce`)이 자판 목록을 만들 때
> 이 테이블을 1회 읽어, 빌트인·외부 자판의 표시명만 id 로 덮어쓴다. 파일이 없으면 XML `<name>`·
> 빌트인 기본명을 그대로 쓴다(무회귀).
